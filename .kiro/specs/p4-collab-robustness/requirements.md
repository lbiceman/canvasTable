# P4 协同编辑健壮性 — 需求文档

## 功能概述

增强 ice-excel 协同编辑的健壮性，覆盖四个核心场景：OT 边界冲突处理、离线编辑与重连同步、协同光标选区优化、多人撤销/重做一致性。

## 用户故事

### US-1：OT 边界测试（合并单元格 + 行列插删 + 公式并发冲突）
**作为** 协同编辑用户，**我希望** 当多人同时对合并单元格区域执行行列插删和公式编辑时，所有客户端最终状态一致，**以便** 避免数据丢失或状态不一致。

### US-2：离线编辑 + 重连同步
**作为** 网络不稳定的用户，**我希望** 断网后仍可继续编辑，重连后本地编辑自动与服务器合并，**以便** 不丢失任何编辑内容。

### US-3：协同光标优化
**作为** 协同编辑用户，**我希望** 看到远程用户选区以半透明背景色高亮显示（而非仅边框），**以便** 更直观地感知其他用户正在编辑的区域。

### US-4：操作历史冲突
**作为** 协同编辑用户，**我希望** 多人同时执行撤销/重做时，各自的操作历史独立且最终状态一致，**以便** 撤销操作不会破坏其他用户的编辑。

## 验收标准

### AC-1：OT 边界
- [ ] 合并单元格区域内的 cellEdit 与 rowInsert/rowDelete 并发时，行索引正确调整
- [ ] 合并单元格区域内的 cellEdit 与 colInsert/colDelete 并发时，列索引正确调整
- [ ] cellMerge 与 rowInsert/colInsert 并发时，合并范围正确扩展/偏移
- [ ] cellMerge 与 rowDelete/colDelete 并发时，合并范围正确收缩或操作被取消
- [ ] 公式单元格（含公式引用）与行列插删并发时，公式引用地址正确调整
- [ ] 所有 OT 转换满足收敛性不变量：apply(apply(S, A), B') === apply(apply(S, B), A')

### AC-2：离线编辑 + 重连同步
- [ ] WebSocket 断开后，本地编辑操作缓存到离线队列
- [ ] 重连后自动发送 join 消息并接收最新文档状态
- [ ] 离线期间的操作通过 OT 转换与服务器状态合并
- [ ] 重连后 OT 客户端修订号与服务器同步
- [ ] 离线队列在重连成功后按序刷新
- [ ] 提供连接状态变化的 UI 反馈（connected/connecting/disconnected）

### AC-3：协同光标优化
- [ ] 远程用户选区绘制半透明背景色填充（alpha 0.15）
- [ ] 背景色使用用户分配的颜色
- [ ] 选区边框保持 2px 实线
- [ ] 用户名标签保持在选区左上角上方
- [ ] 多个远程用户选区可同时显示且不互相遮挡

### AC-4：操作历史冲突
- [ ] 协同模式下每个用户维护独立的撤销/重做栈
- [ ] 撤销操作生成反向操作并通过 OT 通道提交
- [ ] 远程操作不影响本地撤销栈的内容
- [ ] 多人同时撤销时，各自的反向操作通过 OT 转换保持一致性
- [ ] 撤销/重做操作的 OT 转换覆盖所有操作类型组合

## 约束条件

- TypeScript 严格模式，禁止 `any` 类型
- 零运行时依赖
- ES Module
- 类型定义集中在 `src/collaboration/types.ts`
- 不修改已有模块的核心逻辑（仅扩展）
- UI 文本使用简体中文
- 代码注释使用中文
