# P4 协同编辑健壮性 — 性能测试报告

## 分析维度

### 1. 加载性能
🟢 **无需优化**

本次变更不涉及应用初始化或页面加载流程。新增的 `OfflineBuffer` 模块仅在协同模式下按需实例化，不影响非协同模式的加载性能。

### 2. 渲染性能
🟢 **无需优化**

协同光标优化仅修改了 `hexToRgba` 的 alpha 参数（0.08 → 0.15），不增加任何额外的 Canvas 绑定调用或绘制操作。渲染路径完全不变。

### 3. 网络请求
🟡 **建议优化**

- 离线缓冲区在重连后会批量发送缓存的操作，如果离线期间积累了大量操作（>100），可能导致重连后的短暂网络突发。
- 当前实现按序逐条发送，未做批量合并。
- 建议：未来可考虑将同类型操作合并（如多次 cellEdit 同一单元格只保留最后一次），减少重连后的网络负载。

### 4. 内存使用
🟢 **无需优化**

- `OfflineBuffer` 使用简单数组存储操作，每个操作约 200-500 字节。
- 即使离线期间积累 1000 个操作，内存占用也不超过 500KB。
- 重连后 `flush()` 立即释放缓冲区内存。
- 公式引用调整函数使用正则替换，无额外内存分配。

### 5. 包体积影响
🟢 **无需优化**

| 新增/修改文件 | 预估增量 |
|--------------|---------|
| `offline-buffer.ts` | ~1.5KB（压缩后 ~0.6KB） |
| `ot.ts` 公式调整函数 | ~2KB（压缩后 ~0.8KB） |
| `websocket-client.ts` 回调 | ~0.3KB |
| `collaboration-engine.ts` 重连逻辑 | ~0.5KB |
| **总计** | **~4.3KB（压缩后 ~2.2KB）** |

增量极小，对总包体积影响可忽略。

## 总结

本次变更对性能无负面影响。唯一的建议优化点是离线缓冲区在极端场景（长时间离线 + 大量操作）下的网络突发，可在后续迭代中通过操作合并策略优化。
