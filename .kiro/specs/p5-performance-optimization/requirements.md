# P5 性能持续优化 — 需求文档

## 功能概述

ICE Excel 已完成 P0-P3 功能开发，P5 聚焦于性能持续优化，涵盖四个方向：样式密集渲染优化、公式 Worker 批量调度优化、内存占用优化（稀疏存储）、Canvas 离屏渲染（冻结窗格缓存）。

## 用户故事

### US-1：样式密集渲染优化
**作为**处理大面积财务报表的用户，**我希望**在 450+ 全样式单元格场景下渲染帧时间显著低于 16ms，**以便**滚动和编辑操作保持流畅。

### US-2：公式 Worker 批量调度优化
**作为**使用大量公式的用户，**我希望**100 个公式的 Worker 计算时间从 ~106ms 降低至更短，**以便**公式密集型工作表响应更快。

### US-3：内存占用优化（稀疏存储）
**作为**处理大型工作表（10 万行+）的用户，**我希望**空单元格不占用内存，**以便**浏览器不会因内存不足而崩溃。

### US-4：Canvas 离屏渲染（冻结窗格缓存）
**作为**使用冻结窗格的用户，**我希望**冻结区域不需要每帧重绘，**以便**滚动时帧率更高。

## 验收标准

### AC-1：样式密集渲染
- [ ] 渲染管线按样式属性分组绘制，减少 Canvas 上下文状态切换次数
- [ ] 相同字体/颜色/对齐方式的单元格批量绘制
- [ ] 450 个全样式单元格渲染帧时间 ≤ 16ms
- [ ] 不影响现有渲染正确性（合并单元格、条件格式、富文本等）

### AC-2：公式 Worker 批量调度
- [ ] 将当前逐条 `queueMicrotask` 合并改为固定大小分片（chunk）发送
- [ ] 减少主线程与 Worker 之间的 `postMessage` 调用次数
- [ ] 100 个公式 Worker 计算总时间降低 ≥ 30%
- [ ] 不影响现有公式计算正确性（超时、取消、回退机制保持不变）

### AC-3：内存占用优化
- [ ] 数据模型从密集二维数组 `Cell[][]` 改为稀疏存储结构 `Map<string, Cell>`
- [ ] 空单元格不创建 Cell 对象，按需创建
- [ ] `getCell()` 对空单元格返回默认值对象（不分配新内存）
- [ ] 所有现有 API（setCellContent、getMergedCellInfo 等）行为不变
- [ ] 10 万行空工作表内存占用显著降低

### AC-4：Canvas 离屏渲染
- [ ] 冻结行区域使用 OffscreenCanvas 缓存
- [ ] 冻结列区域使用 OffscreenCanvas 缓存
- [ ] 冻结交叉区域使用 OffscreenCanvas 缓存
- [ ] 仅在冻结区域数据变更时重绘缓存，滚动时直接 drawImage
- [ ] 不影响冻结分隔线、选区高亮等视觉效果

## 约束条件

1. 所有代码使用 TypeScript，禁止 `any` 类型
2. 代码注释使用中文，变量名/函数名使用英文
3. 零运行时依赖（不引入新的第三方库）
4. 向后兼容：不破坏任何现有功能和 API
5. 保持 MVC 架构：Model / View / Controller 职责分离
