# P3 数据能力 — 需求文档

## 功能概述

增强 ice-excel 的数据导入/导出能力和数据分析功能，包括 XLSX 导入/导出兼容性提升、大文件流式解析、CSV 编码自动检测、以及数据透视表功能补全。

## 用户故事

### US-1：XLSX 导入兼容性
**作为**用户，**我希望**导入 .xlsx 文件时合并单元格、条件格式、数字格式能被正确还原，**以便**在 ice-excel 中查看和编辑 Excel 文件时不丢失格式信息。

### US-2：XLSX 导出兼容性
**作为**用户，**我希望**从 ice-excel 导出的 .xlsx 文件在 Excel 中打开时格式不丢失，**以便**与使用 Excel 的同事无缝协作。

### US-3：大文件导入
**作为**用户，**我希望**导入 10MB+ 的 XLSX 文件时浏览器不卡死，**以便**处理大型数据集。

### US-4：CSV 导入编码检测
**作为**用户，**我希望**导入 CSV 文件时自动识别 UTF-8、GBK、Shift-JIS 等编码，**以便**无需手动选择编码即可正确显示中日文内容。

### US-5：数据透视表补全
**作为**用户，**我希望**数据透视表功能完整可用，包括排序、多值字段、结果导出，**以便**进行数据分析。

## 验收标准

### AC-1：XLSX 导入兼容性
- [x] 合并单元格正确还原（rowSpan/colSpan/mergeParent）
- [x] 条件格式规则导入（greaterThan/lessThan/between 等基础条件）
- [x] 数字格式导入（货币、百分比、日期、科学计数法）
- [x] 字体样式完整导入（加粗、斜体、下划线、删除线、字号、颜色、字体族）
- [x] 背景色、对齐方式、自动换行正确导入
- [x] 边框样式正确导入（实线、虚线、点线、双线）

### AC-2：XLSX 导出兼容性
- [x] 条件格式规则导出到 XLSX（ExcelJS 条件格式 API）
- [x] 数字格式正确映射到 Excel 格式字符串
- [x] 合并单元格正确导出
- [x] 所有字体样式正确导出
- [x] 边框样式正确导出

### AC-3：大文件导入
- [x] 10MB+ XLSX 文件使用流式/分块解析
- [x] 解析过程中显示进度提示
- [x] 不阻塞浏览器主线程（使用 Web Worker 或分块处理）
- [x] 解析失败时给出友好错误提示

### AC-4：CSV 导入编码检测
- [x] 自动检测 UTF-8（含 BOM）
- [x] 自动检测 GBK/GB2312/GB18030
- [x] 自动检测 Shift-JIS
- [x] 检测失败时回退到 UTF-8
- [x] 支持用户手动选择编码（可选）

### AC-5：数据透视表补全
- [x] 现有引擎（PivotTable）和面板（PivotTablePanel）功能完整
- [x] 支持排序（按行标签或聚合值排序）
- [x] 支持多值字段同时显示
- [x] 结果可写入新工作表
- [x] 面板 UI 交互流畅

## 约束条件

- 零运行时依赖原则：ExcelJS 和 jsPDF 已在 dependencies 中，不新增第三方依赖
- 编码检测使用纯 TypeScript 实现（基于字节特征检测），不引入外部库
- TypeScript 严格模式，禁止 `any`（除非必要并注释）
- 所有新增类型定义集中在 `types.ts` 或模块内部 `types.ts`
- UI 文本使用简体中文
- 代码注释使用中文
