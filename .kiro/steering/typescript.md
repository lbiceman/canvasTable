# TypeScript 编码规范

## 语言要求
- 所有代码必须使用 TypeScript 编写
- 禁止使用纯 JavaScript (.js) 文件
- 禁止使用 `any` 类型，除非绝对必要并添加注释说明原因

## 类型定义
- 优先使用 `interface` 定义对象类型
- 使用 `type` 定义联合类型、交叉类型或类型别名
- 函数参数和返回值必须显式声明类型
- 避免类型断言 (`as`)，优先使用类型守卫

## 代码风格
- 使用 `const` 优先，必要时使用 `let`，禁止 `var`
- 使用箭头函数处理回调
- 使用模板字符串而非字符串拼接
- 使用解构赋值简化代码

## 模块规范
- 使用 ES Module (`import`/`export`)
- 每个文件应有单一职责
- 类型定义集中在 `types.ts` 中
