# 知识库

项目使用 MCP Memory 知识库存储跨项目通用经验。以下是所有实体的索引，按需查询（`mcp_memory_search_nodes` 或 `mcp_memory_open_nodes`），由你判断何时使用哪个。

## 使用原则

1. **查询优先级**：先搜项目代码 → 再搜知识库 → 最后问用户
2. **及时记录且通用化**：完成有价值的工作后写入知识库，**禁止**写入项目特定内容（项目名、具体文件路径、业务逻辑细节），**必须**提炼为跨项目通用的经验、模式、技巧
3. **规则同步**：每次修改 steering 规则文件后，必须同步更新知识库中的对应实体

## 技能 (skill)

| 名称 | 简介 |
|------|------|
| 需求文档驱动开发Skill | 接收需求文档（PPT/XMind/PDF等），走六步流程：解析→生成PRD→代码实现→E2E测试→代码提交→收尾 |
| E2E测试调试方法论 | 截图驱动调试，失败时先截图看页面状态再修，禁止盲改 |
| Skill-中国手机号正则 | 中国大陆手机号的宽松/精确/按运营商正则表达式 |
| TODO批量功能开发经验 | 根据TODO列表批量开发多个功能模块的方法和注意事项 |

## 工作流步骤 (workflow_step)

| 名称 | 简介 |
|------|------|
| PRD-Step1-解析需求文档 | 解析各种格式文档，提取功能模块和业务规则 |
| PRD-Step2-生成需求文档 | 在docs/下生成结构化Markdown需求文档 |
| PRD-Step3-代码实现 | 前后端开发流程、防回归原则 |
| PRD-Step4-E2E测试 | 完整覆盖PRD所有需求点的测试编写策略 |
| PRD-Step5-代码提交 | 测试通过后立即git提交 |
| PRD-Step5-收尾 | 清理临时文件、总结 |

## 原则 (principle)

| 名称 | 简介 |
|------|------|
| PRD-防回归原则 | 新增优于修改、接口兼容、数据兼容、先跑老测试 |
| Memory知识库使用规范 | 查询优先级、记录通用化、禁止写入项目特定内容 |

## 方法论 (methodology)

| 名称 | 简介 |

| 提示词方法论 | 去冗余、强精准、定输出的提示词编写三原则 |
