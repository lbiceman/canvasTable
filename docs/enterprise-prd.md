# ICE Excel 企业版 - 需求文档

> 基于 PRD v1.0 (2026-03-31) 生成

## 1. 项目概述

ICE Excel 企业版定位为安全协作型在线表格，面向对数据安全、权限管控和合规审计有强需求的企业客户（金融、医疗、政府）。在现有 ICE Excel 实时协作编辑能力基础上，集成四大企业级能力：

- 身份认证与单点登录
- 细粒度单元格/工作表保护
- 角色权限控制
- 操作审计日志

### 核心价值主张

"让协作更高效，让安全有保障" — 一次登录 · 多重保护 · 全程可追溯

### 目标用户

| 角色 | 职责 |
|------|------|
| 企业管理员 | 配置安全策略，管理员工权限，审查审计日志 |
| 企业员工 | 在受保护环境下安全协作编辑 |
| 信息安全官 (CISO) | 合规审计、权限治理 |

## 2. 用户认证与授权 (Auth)

### 2.1 支持的认证方式

| 认证方式 | 协议 | 适用场景 | 优先级 |
|----------|------|----------|--------|
| OAuth 2.0 (Google/Microsoft/飞书) | OAuth 2.0 / OIDC | 已使用 SSO 的企业 | P0 |
| 企业 SSO (SAML 2.0) | SAML 2.0 | 大型企业 / Okta / Azure AD | P0 |
| 邮箱 + 密码 (本地) | BCrypt + salt | 无 SSO 的小团队 | P1 |
| LDAP / Active Directory | LDAP / Kerberos | 本地部署企业 | P2 |

### 2.2 登录流程

1. 用户访问 ICE Excel → 跳转企业 IdP → 认证成功 → JWT Token → 自动刷新
2. Token 有效期: 8h
3. 密码错误 5 次后锁定 5 分钟

### 2.3 RBAC 权限模型

| 角色 | 描述 | 典型用户 |
|------|------|----------|
| Super Admin | 管理所有租户/全局安全策略 | IT 主管 |
| Tenant Admin | 管理企业用户和工作表权限 | 部门主管 |
| Member | 创建和编辑授权工作表 | 员工 |
| Guest | 仅查看明确分享的工作表 | 外部合作伙伴 |

## 3. 单元格/工作表保护

### 3.1 保护层级

| 粒度 | 说明 | 典型场景 |
|------|------|----------|
| 工作簿级 | 整个文件加密，需密码打开 | 财务报表 |
| 工作表级 | 禁止编辑特定 Sheet，需密码解锁 | 配置表 |
| 单元格/区域级 | 指定区域锁定，需密码或权限编辑 | 公式单元格/表头行 |
| 行/列级 | 禁止编辑整行或整列 | 序号列 |

### 3.2 保护设置选项

- [x] 编辑单元格
- [ ] 格式化单元格
- [ ] 插入行列
- [x] 撤销/重做
- 设置密码

### 3.3 技术规格

- 加密算法: PBKDF2 (100,000 轮) + AES-256-GCM
- 密码错误 5 次后锁定 5 分钟（防暴力破解）
- 支持无密码保护模式（通过权限系统控制解锁）

## 4. 协同编辑权限控制

### 4.1 权限矩阵

| 角色 | 查看 | 编辑 | 格式化 | 保护设置 | 导出 |
|------|------|------|--------|----------|------|
| Admin | ✅ | ✅ | ✅ | ✅ | ✅ |
| Editor | ✅ | ✅ | ✅ | ❌ | ✅ |
| ReadOnly | ✅ | ❌ | ❌ | ❌ | ✅ |
| Guest | ✅ | ❌ | ❌ | ❌ | ❌ |

### 4.2 权限传播规则

1. 分享链接默认「仅查看」，管理员可升级为「可编辑」
2. 编辑者再分享时，权限不得超过自身角色
3. 管理员可强制实时收回任意用户权限
4. 用户离开企业后自动丧失所有 ICE Excel 访问权限（SCIM 联动）

## 5. 审计日志

### 5.1 必记事件清单

| 类别 | 事件 |
|------|------|
| 认证事件 | LOGIN, LOGIN_FAILED, LOGOUT, TOKEN_REFRESH, PASSWORD_CHANGE, SSO_ASSERTION |
| 权限事件 | PERMISSION_GRANT, PERMISSION_REVOKE, ROLE_CHANGE, SHARE_CREATED, SHARE_REVOKED |
| 数据事件 | CELL_EDIT, CELL_PROTECT, SHEET_LOCK, SHEET_UNLOCK, BULK_UPDATE, DATA_EXPORTED |
| 管理事件 | USER_INVITED, USER_REMOVED, POLICY_CHANGED, WORKBOOK_CREATED |

### 5.2 合规保留策略

- 日志保留 ≥ 180 天（可配置，最长 3 年）
- 防篡改存储: 哈希链 + 企业内私有云
- 支持导出 CSV/JSON 供 SIEM 系统消费（Splunk / ELK / 阿里云 SLS）
- 管理员可设置数据脱敏规则（手机号、身份证等字段自动遮蔽）

## 6. 非功能性需求

| 维度 | 要求 |
|------|------|
| 性能 | 支持 ≥ 50 并发编辑者；P95 延迟 < 200ms |
| 可用性 | SLA ≥ 99.9%；月停机 < 44 分钟；热备切换 |
| 安全 | SOC 2 Type II, ISO 27001, GDPR 合规；私有云部署 |
| 兼容性 | Chrome/Edge/Firefox/Safari 最新 2 个版本；钉钉/企业微信内嵌浏览器 |
| 可扩展性 | 多租户；单租户 ≥ 10,000 用户 |

## 8. 工程实现范围（M1/M2 阶段）

基于现有 ICE Excel 代码库，本次实现聚焦以下核心功能：

### 前端 (TypeScript)
1. **登录页面** - 支持邮箱密码登录 + OAuth/SSO 入口
2. **RBAC 权限控制** - 基于角色的 UI 控制（Admin/Editor/ReadOnly/Guest）
3. **工作表保护** - 保护设置面板、锁定单元格渲染、编辑拦截
4. **权限面板** - 在线用户列表、角色显示、权限设置
5. **审计日志面板** - 日志列表、筛选、导出

### 后端 (Java Spring Boot)
1. **认证模块** - JWT Token 签发/验证、登录/登出 API
2. **用户管理** - 用户 CRUD、角色分配
3. **权限服务** - 房间级权限检查、权限变更 API
4. **工作表保护** - 保护状态持久化、密码验证
5. **审计日志** - 事件记录、查询、导出 API
